
Contributing Guidelines

Thank you for considering contributing to this Industrial IoT Data Pipeline project!

How to Contribute

1. Fork the repository and clone your fork.
2. Create a new branch for your feature or fix.
3. Make your changes with clear commit messages.
4. Test thoroughly before pushing.
5. Submit a pull request with a description of your changes.

Code Guidelines

- Follow the existing code style and structure.
- Use meaningful names for variables and functions.
- Keep functions focused and modular.
- Include comments where necessary, especially for complex logic.

Issues and Bugs

- If you encounter a bug, open an issue with steps to reproduce it.
- Suggestions for improvements are also welcome.

License
